## 분석

이 발췌가 수식 글쓰기의 정전(canon)으로 통하는 이유 하나. 수식 그 자체가 동일하다. §2와 §3의 두 버전은 정확히 같은 정의, 같은 정리, 같은 증명 단계를 다룬다. 다른 것은 오직 문장 결합 방식, 기호 배치, 서술 흐름뿐이다. 두 버전을 나란히 놓고 보면 "수식이 맞으면 글도 괜찮다"는 흔한 오해가 한 페이지에서 무너진다. 후배가 자기 증명 초고를 들고 와 "수식은 다 검증했다"라 말할 때, 이 발췌를 펼쳐 같은 수식이 어떻게 두 가지 글이 될 수 있는지 보여 주면 설득의 시간이 단축된다.

§2 버전이 atrocious한 첫째 이유는 rule 10의 "blah test"를 통과하지 못한다는 점이다. 모든 displayed equation을 단어 "blah"로 치환하고 산문만 읽었을 때 §2의 문장은 끊긴다. 수식이 문장의 문법적 구성요소가 아니라 삽입된 객체로 떠 있기 때문이다. 가령 정의를 도입하는 문장이 콜론으로 끝나고 displayed equation이 등장한 뒤 다음 문장이 새 주어로 시작하면, 수식을 빼고 읽었을 때 "Let X be defined as: . The function X then …"처럼 분절된다. §3 버전은 동일한 displayed equation을 문장의 보어 자리에 놓아, 수식을 빼더라도 "Let X be defined as blah, so that the function X then …"처럼 연속된다. 같은 수학이 같은 페이지에서 두 가지 산문이 된다.

둘째 이유는 rule 4, 정리 진술의 자기완결성이다. §2의 정리는 앞 단락에서 정의한 기호 $S_q$, $p$ 등을 재인용 없이 그대로 사용한다. 독자는 정리만 읽고서는 $S_q$가 무엇인지, $p$가 어떤 범위인지 알 수 없어 앞으로 돌아가야 한다. §3의 정리는 같은 기호를 쓰되 진술 안에 "where $q < p$ and $S_q$ denotes …"라는 짧은 어구를 끼워 넣어 자기완결적으로 만든다. 길이는 한 줄 늘어났을 뿐인데 정리만 따로 떼어 읽어도 의미가 닫힌다. 후배가 자기 정리를 점검하는 가장 빠른 방법은 정리 진술만 복사해 새 페이지에 놓고 그 진술이 단독으로 이해되는지 묻는 것이다. 안 된다면 §3 패턴, 곧 짧은 종속절로 모든 기호를 그 자리에서 정의하는 패턴을 적용한다.

셋째 이유는 rule 1, 인접 기호 사이의 단어 삽입이다. §2에서 "$S_q$, $q < p$"처럼 두 수식이 쉼표 하나로 인접하면 독자는 둘이 같은 객체의 두 표현인지 두 개의 독립된 조건인지 즉시 가르지 못한다. §3에서 "$S_q$, where $q < p$"로 단 한 단어 "where"를 삽입하면 모호함이 사라진다. 이 작은 단어 삽입이 가독성에 미치는 효과는 비대칭적으로 크다. 토큰 한 개의 비용으로 해석 분기를 닫는다. Knuth가 27 rules 중 첫째로 이 규칙을 둔 데에는 이유가 있다.

같은 단락 안에서 동시에 발효되는 rule이 여럿이라는 점이 이 발췌의 교육적 효율을 높인다. rule 1(인접 기호 분리), rule 2(기호로 문장 시작 금지), rule 4(정리 자기완결성), rule 6(displayed equation의 번호 부여 기준), rule 9(기호 일관성), rule 10(blah test), rule 14(기호 정렬), rule 16("such that" vs "where" 선택)가 한 단락 rewrite 안에서 연쇄적으로 걸린다. 한 사례에서 여러 원칙을 한꺼번에 익히면 후배의 작업 비용이 크게 줄어든다. 자기 증명 한 단락을 §3 패턴으로 다시 쓰는 것이 곧 여덟 가지 rule을 동시에 점검하는 행위가 된다.

한국어 수학 글쓰기로 옮기면 위치 문제 하나가 추가된다. 영어의 "where", "such that"은 종속절을 뒤에서 끌어와 진술의 자기완결성을 채운다. 한국어는 SOV 어순이므로 종속절이 앞에서 와야 자연스럽다("$q < p$인 $S_q$에 대해, …"처럼). "단,"이나 "여기서"는 종속절 뒤에 추가 조건을 붙일 때 쓰는 어구지만, 정리 진술 안의 기호 정의는 한국어에서 영어처럼 뒤로 미루기보다 앞에 두는 편이 자기완결성을 더 잘 확보한다. 이 차이를 인지하지 못한 채 영어 rewrite 패턴을 그대로 한국어로 옮기면 어색해진다.

## 따라 쓰기 패턴

이 패턴은 자기 증명 한 단락을 점검하는 3 체크다. ① 모든 displayed equation을 "blah"로 치환한 문장 골격을 따로 적어 본다 — 산문이 흐르는가, 끊기는가. ② 정리 진술만 복사해 새 페이지에 놓고 단독으로 읽는다 — 모든 기호가 그 자리에서 정의되는가. ③ 인접한 두 수식 사이에 단어가 있는가 — "$x$, $y > 0$"이 아니라 "$x$, where $y > 0$"인가. 세 점검 중 하나라도 실패하면 §3 버전의 문장 결합 패턴을 가져와 재배치한다.

한국어 증명 단락을 쓸 때는 정리 진술의 기호 정의 어구를 진술 앞에 두는 패턴이 자기완결성에 유리하다는 점을 기억해 둔다. "정리. $f$가 연속이고 $K$가 컴팩트일 때, $f$는 $K$ 위에서 균등연속이다"처럼 조건이 앞에 와야 한국어 어순이 살고, 동시에 진술이 단독으로 닫힌다.
